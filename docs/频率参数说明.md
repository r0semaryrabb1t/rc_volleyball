# 频率参数说明

本项目涉及多种频率参数，分布在不同节点和配置文件中，各有不同作用。

---

## 1. 控制循环频率 (`control_frequency`)

| 节点 | 默认值 | 配置文件 |
|------|--------|----------|
| `motor_control_node` | 200 Hz | `config/control_params.yaml` |
| `chassis_control_node` | 100 Hz | `config/chassis_params.yaml` |
| `omni_chassis_control_node` | 100 Hz | `config/omni_chassis_params.yaml` |
| `delta_arm_node` | 200 Hz | `config/delta_arm.yaml` |

**作用**: 定时器每隔 `1/freq` 秒触发一次控制回调，在回调中完成：

- 读取最新命令 → 打包发送给电机
- 接收电机反馈 → 发布状态话题

**影响**:

- 频率越高 → 控制越实时、运动越平滑
- 频率过高 → CPU 占用上升、总线带宽可能饱和
- 频率过低 → 响应延迟增大、振荡风险增加

**代码示例** (`motor_control_node.cpp`):

```cpp
double control_freq = this->get_parameter("control_frequency").as_double();
control_timer_ = this->create_wall_timer(
    std::chrono::microseconds(static_cast<int>(1e6 / control_freq)),
    std::bind(&MotorControlNode::controlLoop, this));
```

---

## 2. CAN 发送频率 (`can_tx_frequency`)

- 在 `motor_control_node` 中统计实际 CAN 帧发送速率
- 通过 `ControlFrequency` 话题发布

**作用**: 监控 CAN 总线实际负载，避免总线拥堵导致丢包。

**消息定义** (`msg/ControlFrequency.msg`):

```
std_msgs/Header header
float64 control_frequency    # 实际控制循环频率 (Hz)
float64 can_tx_frequency     # CAN 发送频率 (Hz)
float64 target_frequency     # 目标控制频率 (Hz)
```

---

## 3. 发布频率 (`publish_frequency` / `publish_rate`)

| 节点 | 默认值 | 配置文件 |
|------|--------|----------|
| `remote_control_node` | 50 Hz | `config/remote_control_params.yaml` |
| `joystick_control_node` | 50 Hz | `config/joystick_params.yaml` |

**作用**: 遥控器 / 手柄以此频率向下游发布速度命令话题。50 Hz 对人工遥控已足够平滑。

---

## 4. 读取频率 (`read_frequency`)

| 节点 | 默认值 | 配置文件 |
|------|--------|----------|
| `ops9_encoder_node` | 100 Hz | `config/encoder.yaml` |

**作用**: 定时轮询串口，读取 OPS9 编码器的位姿数据 (X/Y/yaw/az)。

---

## 5. 波特率 (`baudrate`)

| 接口类型 | 默认波特率 | 说明 |
|----------|-----------|------|
| CAN 接口 | 921600 bps | DJI / 达妙电机通信 |
| 串口（宇树） | 4000000 bps | GO-M8010-6 高速串口 |
| OPS9 编码器 | 115200 bps | ASCII 协议串口 |

**作用**: 物理层通信速率，必须与硬件设备端设置一致，否则通信失败。

---

## 6. 监控刷新频率

| 节点 | 刷新率 | 说明 |
|------|--------|------|
| `motor_monitor_node` | 100 Hz (10ms) | 终端状态面板刷新 |
| `delta_arm_monitor_node` | 定时器驱动 | Delta 臂状态显示 |

**作用**: 刷新终端 UI 显示，同时对比实际频率与目标频率进行健康检测：

- 实际 >= 目标 × 95% → 绿色（正常）
- 实际 >= 目标 × 80% → 黄色（警告）
- 实际 < 目标 × 80% → 红色（异常）

---

## 7. 硬件层轮询频率（内部固定值）

| 位置 | 频率 | 说明 |
|------|------|------|
| `can_interface.cpp` 接收线程 | ~10 kHz (100us) | CAN 帧高频轮询接收 |
| `serial_interface.cpp` 累积接收 | ~2 kHz (500us) | 串口数据轮询接收 |

**作用**: 硬件层内部轮询间隔，确保不丢失总线上的数据帧，同时避免 CPU 空转。

---

## 总结

```
用户输入 (50Hz)          控制层 (100~200Hz)         硬件层 (2k~10kHz)
┌──────────────┐      ┌──────────────────┐      ┌─────────────────┐
│ 遥控器/手柄   │─────▶│ 底盘/机械臂控制   │─────▶│ CAN/串口 收发    │
│ publish_rate │      │ control_frequency│      │ 内部轮询         │
└──────────────┘      └──────────────────┘      └─────────────────┘
                              │
                              ▼
                      ┌──────────────────┐
                      │ monitor 显示      │
                      │ 100Hz 刷新        │
                      └──────────────────┘
```

频率从上层到底层逐级提高：用户输入不需要太快（50Hz），控制层需要较高频率保证平滑（100~200Hz），硬件层需要最高轮询频率避免丢帧（kHz 级）。
