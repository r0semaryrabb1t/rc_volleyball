# 底盘控制节点数据流分析

## 1. 系统拓扑

```
                          /cmd_vel (geometry_msgs/Twist)
                              │  vx, vy, wz
                              ▼
                   ┌───────────────────────┐
                   │  chassis_control_node  │  配置: chassis_params.yaml
                   │                        │  ├─ 底盘几何 (轮距/轴距/轮径)
                   │  ① 逆运动学解算        │  ├─ 速度限制
                   │  ② 舵角最短路径优化     │  ├─ 电机名称映射 (8 个电机)
                   │  ③ 机械角→编码器角变换  │  └─ steer_offset / direction
                   └───────┬────────────────┘
                           │                         ▲
          publish (×8 msg) │                         │ subscribe
                           ▼                         │
               /dji_motor_command_advanced    /dji_motor_states
               (DJIMotorCommandAdvanced)     (DJIMotorState)
                           │                         ▲
                           ▼                         │
                   ┌───────────────────────┐         │
                   │  motor_control_node    │  配置: motors.yaml + pid_params.yaml
                   │                        │  ├─ device, baudrate, motor type/id
                   │  DJI PID 闭环          │  └─ PID 参数 (Kp, Ki, Kd)
                   │  CAN SendRecv          │────────┘
                   └───────────────────────┘
                           │
                      CAN Bus
                    ┌──────┴──────┐
              /dev/ttyACM0   /dev/ttyACM2
              4×GM6020(转向)  4×GM3508(驱动)
```

## 2. 详细数据流

### 2.1 输入: /cmd_vel → chassis_control_node

```
geometry_msgs/Twist
  linear.x  → cmd_vx_   (clamp to ±max_linear_velocity)
  linear.y  → cmd_vy_   (clamp to ±max_linear_velocity)
  angular.z → cmd_wz_   (clamp to ±max_angular_velocity)
```

- **超时保护**: 500ms 无新命令 → vx/vy/wz 全部归零
- **死区阈值**: 0.01 m/s / 0.01 rad/s 以下视为无运动

### 2.2 控制循环 (100 Hz)

```
每个周期:
  if 无运动命令:
      4 个舵轮全部 → angle=0°(机械零位), velocity=0
  else:
      Step 1: 逆运动学
          (vx, vy, wz) → 4× WheelCommand(angle°, velocity m/s)

      Step 2: 舵角优化 (每个轮独立)
          current_mech = encoder_angle - steer_offset     # 编码器→机械
          target_mech  = kinematic_angle × steer_direction # 运动学→机械
          optimized    = optimizeSteerAngle(current, target, velocity)
          // 若 |Δ角| > 90°: 反转驱动速度, 舵角 ±180°

      Step 3: 发布命令 (每轮 2 条消息)
```

### 2.3 输出: chassis_control_node → motor_control_node

每个舵轮单元发送 **2 条** `DJIMotorCommandAdvanced` 消息:

#### 转向电机 (GM6020 — 位置控制)

```
joint_name      = "DJI6020_1" (etc.)
mode            = MODE_POSITION (2)
position_target = (mechanical_angle + steer_offset) × π/180   [rad]
```

代码路径: `chassis_control_node.cpp:260-262`
```cpp
double encoder_angle_deg = cmd.angle + steer_offset;
steer_msg.position_target = encoder_angle_deg * M_PI / 180.0;
```

#### 驱动电机 (GM3508 — 速度控制)

```
joint_name      = "DJI3508_1" (etc.)
mode            = MODE_VELOCITY (1)
velocity_target = wheel_velocity / (2π × wheel_radius) × 60 × (π/30) × drive_direction  [rad/s]
                  \___________ m/s → RPM ___________/   \_ RPM→rad/s _/   \_ 方向 _/
```

代码路径: `chassis_control_node.cpp:266-273`
```cpp
double rpm = (cmd.velocity / (2.0 * M_PI * wheel_radius_)) * 60.0;
drive_msg.velocity_target = rpm * M_PI / 30.0 * drive_direction;
```

### 2.4 motor_control_node 接收处理

`djiCommandAdvancedCallback()` (`motor_control_node.cpp:690`):

```
查找 motor by joint_name → motors_ map
switch (mode):
  MODE_POSITION:
    setPositionTarget(position_target × 180/π)     // rad → deg
    → 位置环 PID → 输出电流
  MODE_VELOCITY:
    setVelocityTarget(velocity_target × 60/(2π))   // rad/s → RPM
    → 速度环 PID → 输出电流
```

PID 参数来源: `pid_params.yaml` (motor_control_node 启动时加载)

### 2.5 反馈: motor_control_node → chassis_control_node

```
motor_control_node publishStates() (每控制周期):
    DJIMotorState {
        joint_name: "DJI6020_1"
        angle: 编码器角度 (度, 0-360)     ← 编码器原始值
        rpm:   转速
        online: 心跳检测结果
    }

chassis_control_node motorStateCallback():
    motor_states_[joint_name] = *msg       ← 缓存到 map
    → 用于舵角优化和正运动学
```

## 3. 坐标变换链

```
    编码器角度 (encoder, 0-360°)
         │
         │  − steer_offset
         ▼
    机械角度 (mechanical, 0°=正前方)
         │
         │  × steer_direction
         ▼
    运动学角度 (kinematic, 逆解算输出)
```

| 变量 | 含义 | 来源 |
|------|------|------|
| `steer_offset` | 机械零位（正前方）时的编码器读数 | chassis_params.yaml, 每轮不同 |
| `steer_direction` | 编码器旋转方向与运动学正方向的映射 | chassis_params.yaml, 1 或 -1 |

## 4. 舵角优化算法

`SteerWheelKinematics::optimizeSteerAngle()`:

```
输入: current_angle (机械角), target_angle (机械角), velocity
  delta = angleDifference(target, current)    // (-180°, 180°]
  if |delta| > 90°:
      target += 180°        // 舵轮少转
      velocity = -velocity   // 驱动反向
  return normalizeAngle(target)
```

**作用**: 避免舵轮旋转超过 90°，改为反向驱动 + 较小角度旋转。

## 5. 配置文件归属

### chassis_params.yaml 独占参数

| 参数 | 示例值 | 说明 |
|------|--------|------|
| `control_frequency` | 100.0 Hz | 底盘控制频率 |
| `wheel_base_x` | 0.65 m | 前后轮距 |
| `wheel_base_y` | 0.62 m | 左右轴距 |
| `wheel_radius` | 0.055 m | 轮径 |
| `fl_steer_motor` | "DJI6020_1" | 电机名称映射 |
| `fl_steer_offset` | 330.8° | 转向零位编码器角度 |
| `fl_steer_direction` | -1 | 舵轮编码器方向 |
| `fl_drive_direction` | 1 | 驱动正方向 |

### motors.yaml 独占参数

| 参数 | 说明 |
|------|------|
| `device` / `baudrate` | CAN 设备路径与波特率 |
| `type` / `id` | 电机型号与总线 ID |

### pid_params.yaml 独占参数

| 参数 | 说明 |
|------|------|
| `GM6020.position_pid` | 转向位置环 PID |
| `GM3508.velocity_pid` | 驱动速度环 PID |

**结论: 三个配置文件职责完全不交叉。**

## 6. 电机映射表

| 舵轮 | 转向 (GM6020) | 驱动 (GM3508) | Offset | Steer Dir | Drive Dir |
|------|:------------:|:------------:|:------:|:---------:|:---------:|
| FL 左前 | DJI6020_1 | DJI3508_1 | 330.8° | -1 | 1 |
| FR 右前 | DJI6020_2 | DJI3508_2 | 89.6° | -1 | -1 |
| RL 左后 | DJI6020_3 | DJI3508_3 | 207.1° | -1 | 1 |
| RR 右后 | DJI6020_4 | DJI3508_4 | 33.0° | -1 | -1 |

- **steer_direction 全 -1**: 四个 GM6020 编码器方向一致，均与运动学约定相反
- **drive_direction 左右对称**: 左侧 +1（正转向前），右侧 -1（安装镜像）

## 7. Topic 一览

| Topic | 方向 | 消息类型 | 频率 | 内容 |
|-------|------|---------|------|------|
| `/cmd_vel` | 外部 → 底盘 | geometry_msgs/Twist | 不定 | vx, vy, wz |
| `/dji_motor_command_advanced` | 底盘 → 电机 | DJIMotorCommandAdvanced | 100Hz × 8 | 位置/速度目标 |
| `/dji_motor_states` | 电机 → 底盘 | DJIMotorState | 200Hz × 8 | 角度/转速/在线 |
| `/chassis_state` | 底盘 → 外部 | ChassisState | 100Hz | 里程计 + 4 舵轮状态 |

## 8. 与 Delta 臂控制模式对比

| 方面 | 底盘 (DJI) | Delta 臂 (Unitree GO8010) |
|------|-----------|--------------------------|
| 控制接口 | CAN 总线 | RS-485 串口 |
| 控制模式 | PID 闭环 (motor_control_node 内部) | FOC 直通 (kp/kd 从 topic 传入) |
| 高层节点职责 | 只发位置/速度**目标** | 发 FOC 全参数 (pos, vel, kp, kd, τ_ff) |
| offset 位置 | chassis_params.yaml (度) | motors.yaml (弧度) |
| 参数覆盖 | **无** — 各配置文件职责不交叉 | kp/kd 每周期被 delta_arm_node 覆盖 |
| direction | chassis_params.yaml 独占 | 两处都有，需手动同步 |

**设计差异根因**: DJI 电机有独立 PID 闭环，高层只需给目标值；宇树 GO8010 是开放式 FOC，控制参数必须由上层每周期提供。
